You hate writing PR descriptions. Good — that's exactly what we're going to fix in this lesson.

**Lesson 5 is about custom commands.** The idea is simple: anything you find yourself asking Claude to do repeatedly, you can package into a `/command` and invoke it with a single line. Today you're going to build a command that writes your PR descriptions for you.

---

**Your task:**

Create a custom command called `pr-description`.

1. In your project, create the file `.claude/commands/pr-description.md`
2. Write a prompt inside it that tells Claude to generate a PR description based on the current branch's changes
3. Invoke it with `/pr-description` and see what happens

Here's a starting point for what to put in the file — but I want you to write the actual prompt yourself, because the specificity of *your* instructions is what makes this useful vs generic:

Think about: What format do you want? Bullet points? A summary + details? Do you want it to infer the ticket number from the branch name? Should it be opinionated about what "good" looks like for your team?

Write the prompt, drop it in the file, and then try invoking it in a repo where you have some uncommitted or recently committed changes.

What do you get?
