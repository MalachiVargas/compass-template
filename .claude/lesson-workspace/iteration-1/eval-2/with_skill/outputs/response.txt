What Happened Exactly — A Debrief

You asked Claude "what does this project do?" and gave it no code. Here's what Claude actually did under the hood, and why it worked the way it did.

---

WHAT YOU SAW

Claude used two tools: Glob and Read.

Glob is a file-pattern matcher. When Claude called it, it was asking: "give me all the files in this directory that match a pattern" — things like **/*.py, *.json, requirements.txt, and so on. It's the equivalent of you opening a terminal and running `ls -R` or `find . -name "*.py"`, but with structure Claude can reason about.

Read is exactly what it sounds like. Once Glob surfaced a list of candidate files, Claude picked the most informative ones and read them — the entry point (main.py or app.py), the config files, maybe a README if one existed.

It didn't read everything. It read strategically.

---

WHY IT STARTED THERE

Your prompt was the signal. "What does this project do?" is a high-level question, so Claude started high-level: project structure first, then the files most likely to explain purpose — entry points, configuration, imports at the top of key files.

If you had asked "where does the portfolio data get fetched?", Claude would have started differently — probably Grep for keywords like "fetch", "request", "api", or whatever your data source is. The question shapes the navigation.

---

THE MENTAL MODEL SHIFT

Here's what this means for how you work with Claude going forward:

Before understanding this, most developers do one of two things:
1. Paste a wall of code into the prompt and say "fix this"
2. Try to explain the whole codebase in words before asking a question

Both are slower and less accurate than letting Claude navigate. When you paste code, you're giving Claude unstructured noise with no file context, no line numbers, no sense of what's nearby. When Claude reads the file itself, it gets structure — and it can follow imports, jump to related files, and cross-reference.

Your five years of Python means you've probably already developed the instinct of knowing which file to open first when debugging. Claude does the same thing — it just does it through Glob and Read instead of muscle memory.

---

THE BIGGER PICTURE

What you watched is the foundation of everything in Claude Code. It's not autocomplete, and it's not a search engine. It's an agent that navigates — the same way you navigate a codebase, just with different tools.

Every more advanced thing you'll do with Claude — multi-step tasks, keeping it focused, building automations — all of it assumes this mental model: Claude is exploring, not reading your mind. You give it a starting point, and it finds its way.

For your finance dashboard specifically: next time you want Claude's help, try giving it a file path and a specific question instead of pasting code. Something like "look at dashboard/data_loader.py — why is the portfolio calculation slow?" You'll notice the difference immediately.
